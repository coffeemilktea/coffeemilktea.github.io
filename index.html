The toolkitSix tools, zero setup

Everything you need to debug, test, and learn HL7 v2.x and DICOM — each tool is a standalone page that works offline once loaded.

  • Medical Imaging

    DICOM Viewer & Tag Morph

    View DICOM studies across every major storage SOP class — CT, MR, US, XA, NM, PET, mammography, RT and more — with window/level, cine playback, and measurements. Edit tags inline or in batch and re-export valid DICOM.

    Create, configure, and generate DICOM files from scratch, then inspect the result with the built-in tag reader. Ideal for testing DICOM implementations, building synthetic datasets, and learning the standard.

  • Clinical Messaging

    HL7 v2.x Parser

    Paste any HL7 v2.x message for an instant segment-by-segment, field-by-field breakdown with names and descriptions from the standard. Understand your interface data at a glance.

AboutBuilt for healthcare integration work

These tools grew out of day-to-day interface engineering: debugging HL7 feeds, testing PACS and modality connections, and building synthetic data for demos and development. They're made for healthcare IT professionals, integration engineers, and students working with medical data standards.

Everything is plain HTML and JavaScript served as static files — there is no backend, no analytics, and no tracking. Because real messages and images often contain PHI, the tools are deliberately designed so that data never leaves the page.

Your data stays on your machine

Every tool runs fully client-side. DICOM files, HL7 messages, and worklist queries are processed in your browser's memory and are never uploaded, logged, or stored anywhere.
